# 実験条件（固定）

今後の4択法令QA実験・研究で共通にする条件をまとめます。

## コア条件
- 対象データ: デジタル庁 4択法令データ `datasets/lawqa_jp/data/selection.json`（全140問）
- インデックス: 全件ベクトルインデックス `data/faiss_index_full`（BM25は使用しない）
- Retriever: `RETRIEVER_TYPE=vector`, `RETRIEVER_TOP_K=10`
- LLM: `Qwen3:8b`（Ollama）

## 実行手順（標準）
```bash
cd ~/yokote/statutes-rags-master
source setup/restore_env.sh
./scripts/evaluate.sh 140
```

## 確認ポイント
- `.env` で `VECTOR_STORE_PATH=data/faiss_index_full` になっていること
- 実行前に `echo $VECTOR_STORE_PATH` と `echo $LLM_MODEL` で念のため確認
- 結果ファイル: `results/evaluations/evaluation_results_final.json`

## 備考
- インデックスは11GB程度あるのでストレージ残量に注意
- スクリプトに実行権がない場合は `bash scripts/evaluate.sh 140` で実行
